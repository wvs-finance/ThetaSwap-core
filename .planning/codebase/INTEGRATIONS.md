# External Integrations

**Analysis Date:** 2026-03-17

## APIs & External Services

**Blockchain RPC Providers:**
- Alchemy
  - What it's used for: JSON-RPC endpoint for mainnet, Sepolia, Unichain testnet/mainnet
  - Client: httpx (Python), cast (Foundry CLI)
  - Auth: `ALCHEMY_API_KEY` env var, passed in URL query param
  - Endpoints:
    - mainnet: `https://eth-mainnet.g.alchemy.com/v2/${ALCHEMY_API_KEY}`
    - sepolia: `https://eth-sepolia.g.alchemy.com/v2/${ALCHEMY_API_KEY}`
    - unichain: `https://unichain-mainnet.g.alchemy.com/v2/${ALCHEMY_API_KEY}`
    - unichain_sepolia: `https://unichain-sepolia.g.alchemy.com/v2/${ALCHEMY_API_KEY}`

- Reactive Network (custom RPC)
  - What it's used for: Reactive hook callback execution and state reading
  - Client: Cast (CLI) for fork tests, custom JSON-RPC via foundry.toml
  - Auth: `REACTIVE_RPC_URL` env var (full URL including credentials if needed)
  - Usage: `test/fee-concentration-index-v2/protocols/uniswap-v3/integration/` fork tests

**Block Explorer APIs:**
- Etherscan
  - What it's used for: Contract verification on mainnet and Sepolia
  - Client: Foundry verify command
  - Auth: `ETHERSCAN_API_KEY` env var
  - Config location: `foundry.toml` lines 57-61
  - Chains: mainnet (chain 1), Sepolia (chain 11155111)

- BlockScout
  - What it's used for: Contract verification on Unichain (mainnet + Sepolia)
  - Client: Foundry verify command
  - Auth: `ETHERSCAN_API_KEY` env var (reused for BlockScout)
  - URLs:
    - unichain: `https://unichain.blockscout.com/api`
    - unichain_sepolia: `https://unichain-sepolia.blockscout.com/api`
  - Config location: `foundry.toml` lines 57-61

**The Graph:**
- Uniswap V3 Subgraph
  - What it's used for: Pool discovery, TVL ranking, volume analytics for cross-pool research
  - Client: httpx (Python), async HTTP POST requests
  - Auth: `GRAPH_API_KEY` env var (required)
  - Gateway URL: `https://gateway.thegraph.com/api/{key}/subgraphs/id/5zvR82QoaXYFyDEKLZ9t6v9adgnptxYpKpSbxtgVENFV`
  - Implementation: `research/econometrics/cross_pool/subgraph.py` (lines 1-50)
  - Query: GraphQL query for top 100 pools ordered by TVL (frozen data in `research/econometrics/cross_pool/data.py`, not live during tests)
  - Usage: Research/analysis only, not called during normal contract operation

## Webhooks & Callbacks

**Incoming:**
- Uniswap V4 Hook callbacks:
  - `afterAddLiquidity` - Called by PoolManager after LP position creation
  - `beforeSwap` - Called before swap execution
  - `afterSwap` - Called after swap execution
  - `afterRemoveLiquidity` - Called after position removal
  - Implementation: `src/fee-concentration-index-v2/FeeConcentrationIndexV2.sol`

- Reactive callback:
  - Reactive network invokes `callback()` on contracts monitoring V3 events
  - Router: `src/reactive-integration/adapters/uniswapV3/V3CallbackRouter.sol`
  - Payload: V3 pool address, tick, liquidity changes (decoded in callback)
  - Gas limit: 1M (increased from 300K to handle 4 staticcalls + SSTOREs)
  - Proxy behavior: On OOG/revert, emits `CallbackFailure` but still calls `pay()`

**Outgoing:**
- None detected - No outbound webhooks to external services

## External Contract Interactions

**Uniswap V4:**
- Contract: PoolManager (immutable, core protocol)
- Methods called:
  - `unlock()` - To execute atomic swaps/liquidity operations
  - State reads: Pool state, tick data, liquidity snapshots

**Uniswap V3:**
- Contract: IUniswapV3Pool (read-only for V3 reactive integration)
- Methods called:
  - `positions()` - Read LP position data
  - `swap()` - Not called, only observed via reactive callbacks
  - Ticks: Read via `liquidity()` and tick observables
  - Implementation: `src/reactive-integration/adapters/uniswapV3/V3CallbackRouter.sol`

**Permit2:**
- Contract: Permit2 (from v4-periphery)
- Purpose: Token approval management via permits
- Location: `lib/v4-periphery/lib/permit2/`
