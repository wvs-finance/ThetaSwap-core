2. `FCI Hook`: registers pool in storage (poolId mapped to protocol flags), initializes A_T = 0, N = 0
3. Note over FCI Hook: "Pool is now tracked. Subsequent hook callbacks will compute FCI terms."

**Flow — Part 2: Swap representative (from ARCHITECTURE.md data flow):**
4. Note: "On each swap..."
5. `FCI Hook` receives `afterSwap(sender, key, params, hookData)` from PoolManager (or UniswapV3Callback)
6. `FCI Hook` extracts `protocolFlags` from `hookData[0:2]`
7. `FCI Hook` ->> `Protocol Facet`: `delegatecall tloadTick(hookData)` -- loads tickBefore from transient storage
8. `FCI Hook` ->> `Protocol Facet`: `delegatecall currentTick(hookData, poolId)` -- reads tickAfter from pool
9. `FCI Hook`: computes tick overlap interval, FCI term (xk), updates A_T accumulator
10. `FCI Hook` ->> `Protocol Facet`: `delegatecall addStateTerm(hookData, poolId, blockLifetime, xSquaredQ128)`
11. `FCI Hook`: emits `FCITermAccumulated` event

**Flow — Part 3: External query:**
12. Note: "Later..."
13. `External Reader` ->> `FCI Hook`: `getDeltaPlus(poolId)` query
14. `FCI Hook` -->> `External Reader`: returns DeltaPlus value
