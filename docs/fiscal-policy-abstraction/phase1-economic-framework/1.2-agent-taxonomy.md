# 1.2 Economic Agent Classification

## Objective
Design flexible agent taxonomy beyond JIT/PLP to support arbitrary economic agent definitions

## Current State Analysis

### Existing Agent Types in ParityTax
- **JIT (Just-In-Time) Providers**: Short-term, opportunistic liquidity providers
- **PLP (Persistent Liquidity Providers)**: Long-term, committed liquidity providers

### Limitations of Current Model
1. **Binary Classification**: Only two agent types limits policy flexibility
2. **Hardcoded Behavior**: Agent characteristics are fixed in code
3. **Limited Customization**: Pool deployers cannot define custom agent types
4. **Static Taxonomies**: No support for dynamic or context-dependent classification

## Proposed Agent Classification Framework

### 1. Multi-Dimensional Agent Taxonomy

#### 1.1 Commitment Dimension
**Definition**: Duration and reliability of liquidity provision

**Categories**:
- **Ephemeral** (0-10 blocks): Ultra-short-term, opportunistic
- **Transient** (10-100 blocks): Short-term, tactical
- **Semi-Persistent** (100-1000 blocks): Medium-term, strategic
- **Persistent** (1000+ blocks): Long-term, foundational
- **Permanent** (Indefinite): Protocol-level commitment

**Economic Implications**:
- **Ephemeral**: High elasticity, low commitment, potential for MEV extraction
- **Transient**: Moderate elasticity, some commitment, tactical positioning
- **Semi-Persistent**: Lower elasticity, significant commitment, strategic value
- **Persistent**: Low elasticity, high commitment, foundational value
- **Permanent**: Near-zero elasticity, maximum commitment, public good provision

#### 1.2 Capital Dimension
**Definition**: Size and nature of capital commitment

**Categories**:
- **Micro** (< $1K): Small individual providers
- **Small** ($1K - $10K): Individual or small group providers
- **Medium** ($10K - $100K): Institutional or large individual providers
- **Large** ($100K - $1M): Institutional providers
- **Whale** ($1M+): Major institutional or protocol providers

**Economic Implications**:
- **Micro/Small**: High participation, low individual impact, sensitive to costs
- **Medium**: Moderate participation, moderate impact, cost-sensitive
- **Large/Whale**: Low participation, high impact, less cost-sensitive

#### 1.3 Strategy Dimension
**Definition**: Behavioral patterns and strategic objectives

**Categories**:
- **Arbitrageurs**: Price difference exploitation
- **Market Makers**: Continuous liquidity provision
- **Yield Farmers**: Fee maximization
- **Hedgers**: Risk management
- **Speculators**: Price direction betting
- **MEV Extractors**: Value extraction from transactions
- **Social Providers**: Public good provision

**Economic Implications**:
- **Arbitrageurs**: High efficiency, low commitment, potential externalities
- **Market Makers**: Moderate efficiency, high commitment, positive externalities
- **Yield Farmers**: Variable efficiency, variable commitment, neutral externalities
- **Hedgers**: Low efficiency, high commitment, risk reduction
- **Speculators**: High risk, variable commitment, price discovery
- **MEV Extractors**: High efficiency, low commitment, negative externalities
- **Social Providers**: Low efficiency, high commitment, positive externalities

#### 1.4 Information Dimension
**Definition**: Access to and use of information

**Categories**:
- **Informed**: Access to privileged information
- **Semi-Informed**: Access to some privileged information
- **Public**: Only public information access
- **Uninformed**: Limited information access

**Economic Implications**:
- **Informed**: Potential for unfair advantage, higher efficiency
- **Semi-Informed**: Moderate advantage, moderate efficiency
- **Public**: Fair competition, standard efficiency
- **Uninformed**: Potential disadvantage, lower efficiency

### 2. Dynamic Agent Classification

#### 2.1 Context-Dependent Classification
**Concept**: Agent type can change based on context and behavior

**Examples**:
- **Time-based**: Agent classified differently during different time periods
- **Volume-based**: Classification changes based on transaction volume
- **Market-based**: Classification depends on market conditions
- **Behavior-based**: Classification evolves based on observed behavior

#### 2.2 Composite Agent Types
**Concept**: Agents can have multiple characteristics simultaneously

**Examples**:
- **Informed Arbitrageur**: High information + arbitrage strategy
- **Social Market Maker**: Public good provision + market making
- **Whale Hedger**: Large capital + risk management
- **Micro MEV Extractor**: Small capital + value extraction

### 3. Agent Behavior Modeling

#### 3.1 Elasticity Characteristics
**Definition**: How agent behavior responds to incentives

**Factors**:
- **Price Elasticity**: Response to fee changes
- **Time Elasticity**: Response to commitment requirements
- **Risk Elasticity**: Response to risk changes
- **Information Elasticity**: Response to information availability

#### 3.2 Utility Functions
**Definition**: Mathematical representation of agent preferences

**Components**:
- **Profit Maximization**: Direct financial returns
- **Risk Minimization**: Volatility and downside protection
- **Social Utility**: Public good provision
- **Reputation**: Long-term standing and trust

### 4. Agent Interaction Patterns

#### 4.1 Competitive Interactions
- **Direct Competition**: Same agent types competing for same opportunities
- **Indirect Competition**: Different agent types competing for same resources
- **Cooperative Competition**: Agents working together while competing

#### 4.2 Complementary Interactions
- **Liquidity Provision**: Different agents providing liquidity at different levels
- **Price Discovery**: Different agents contributing to price formation
- **Risk Sharing**: Different agents taking different risk profiles

#### 5.2 Dynamic Classification Rules
```solidity
function classifyAgent(
    address agent,
    AgentMetrics memory metrics
) public view returns (AgentType) {
    // Commitment-based classification
    if (metrics.commitmentDuration < 10) {
        return AgentType.EPHEMERAL;
    } else if (metrics.commitmentDuration < 100) {
        return AgentType.TRANSIENT;
    }
    
    // Capital-based classification
    if (metrics.capitalSize > 1000000) {
        return AgentType.WHALE;
    }
    
    // Strategy-based classification
    if (metrics.mevExtraction > threshold) {
        return AgentType.MEV_EXTRACTOR;
    }
    
    // Composite classification
    return determineCompositeType(metrics);
}
```

### 6. Agent Registry and Management

#### 6.1 Agent Registration
- **Self-Registration**: Agents declare their type and characteristics
- **Observed Classification**: System classifies based on observed behavior
- **Hybrid Approach**: Combination of self-declaration and observation

#### 6.2 Agent Verification
- **Proof of Commitment**: Cryptographic proof of commitment duration
- **Proof of Capital**: Proof of capital size (without revealing exact amount)
- **Proof of Strategy**: Evidence of strategy type
- **Proof of Information**: Proof of information access level

#### 6.3 Agent Evolution
- **Type Transitions**: How agents can change type over time
- **Grace Periods**: Time allowed for type changes
- **Verification Requirements**: What proof is needed for type changes

## Implementation Considerations

### 1. Gas Efficiency
- **Caching**: Cache agent classifications to avoid recalculation
- **Batch Processing**: Process multiple agents simultaneously
- **Lazy Evaluation**: Only classify when needed

### 2. Privacy and Anonymity
- **Zero-Knowledge Proofs**: Prove agent characteristics without revealing identity
- **Partial Disclosure**: Reveal only necessary information
- **Anonymity Sets**: Group agents to maintain privacy

### 3. Governance and Upgrades
- **Agent Type Addition**: How to add new agent types
- **Classification Rule Updates**: How to update classification logic
- **Migration**: How to handle existing agents when rules change

## Economic Implications

### 1. Market Efficiency
- **Specialization**: Different agent types can specialize in different functions
- **Competition**: Multiple agent types can compete in same space
- **Innovation**: New agent types can emerge and be recognized

### 2. Policy Flexibility
- **Targeted Taxation**: Different tax rates for different agent types
- **Behavioral Incentives**: Encourage or discourage specific behaviors
- **Social Goals**: Promote socially beneficial agent types

### 3. System Stability
- **Diversification**: Multiple agent types reduce systemic risk
- **Redundancy**: Different agent types can provide similar functions
- **Resilience**: System can adapt to changes in agent behavior
