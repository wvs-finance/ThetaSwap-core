# Implementation Plan: Reactive FCI for Non-V4 Pools

**Branch**: `003-reactive-integration` | **Date**: 2026-03-06 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `kitty-specs/001-reactive-fci-non-v4-pools/spec.md`

## Summary

Extend FeeConcentrationIndex coverage to Uniswap V3 pools via reactive smart contracts. A ReactiveHookAdapter at `src/reactive-integration/adapters/uniswapV3/` owns a parallel FCI storage instance (REACTIVE_FCI_STORAGE_SLOT) and processes V3 Swap/Mint/Burn/Collect events through 4 separate typed callbacks. Collect fees are accumulated per position over its lifetime; on Burn, the accumulated total drives the fee share ratio computation. The adapter reuses all existing FCI Mod files (TickRangeRegistryMod, AccumulatedHHIMod, FeeShareRatioMod, SwapCountMod, BlockCountMod) without modification. Platform-specific reactive subscription contracts (Reactive Network / Somnia L1) handle event sourcing and are not part of the adapter.

## Technical Context

**Language/Version**: Solidity ^0.8.26
**Primary Dependencies**: Uniswap V4 core (PoolKey, PoolId, Currency), Uniswap V3 core (IUniswapV3Pool, IUniswapV3Factory), Solady (FixedPointMathLib), forge-std, kontrol-cheatcodes
**Storage**: Diamond storage pattern (keccak256 slot hashing). Adapter owns REACTIVE_FCI_STORAGE_SLOT, disjoint from V4 FCI's FCI_STORAGE_SLOT.
**Testing**: Kontrol formal proofs (prove_ prefix) for type-level invariants, Foundry fuzz tests (testFuzz_ prefix) for system-level invariants
**Target Platform**: EVM chains (Ethereum mainnet for V3 pools, Reactive Network / Somnia L1 for event sourcing)
**Project Type**: Single (Solidity smart contracts)
**Constraints**: SCOP (no inheritance except tests, no `library` keyword, no `modifier` keyword). All helpers as file-level free functions in Mod files.

## Remaining Work

### WP-01: ReactiveHookAdapter Storage

Create `ReactiveHookAdapterStorageMod.sol` with:
- `REACTIVE_FCI_STORAGE_SLOT` — parallel FCI state (AccumulatedHHI, TickRangeRegistry, feeGrowthBaseline per pool)
- `COLLECT_FEE_STORAGE_SLOT` — accumulated Collect fees per position: `mapping(PoolId => mapping(bytes32 => CollectedFees))`
- `CollectedFees` struct: `{ uint256 amount0, uint256 amount1 }`
- Storage accessor free functions

### WP-02: ReactiveHookAdapter Contract

Create `ReactiveHookAdapter.sol` at `src/reactive-integration/adapters/uniswapV3/` with:
- **State**: `mapping(address => bool) authorizedCallers`, `address owner`, immutable `IUniswapV3Factory factory`
- **Auth**: inline `if (!authorizedCallers[msg.sender]) revert Unauthorized()` on all callbacks
- **`onV3Swap(V3SwapData)`**: construct synthetic PoolKey via `fromV3Pool()`, call `incrementOverlappingRanges(poolId, tick, tick)` on own storage
- **`onV3Mint(V3MintData)`**: construct PoolKey, derive positionKey, call `registry.register()`, store fee baseline (zero — Collect events will supply fees)
- **`onV3Burn(V3BurnData)`**: construct PoolKey, derive positionKey, read accumulated CollectedFees, compute `SyntheticFeeGrowth.fromBurnAmount()`, compute `FeeShareRatio`, call `registry.deregister()`, call `accumulatedHHI.addTerm()`, clean up CollectedFees
- **`onV3Collect(V3CollectData)`**: construct PoolKey, derive positionKey, accumulate `feeAmount0`/`feeAmount1` into `CollectedFees`
- **`getIndex(PoolKey)`**: read own AccumulatedHHI, return (indexA, indexB)
- **`setAuthorized(address, bool)`**: owner-only whitelist management

### WP-03: Fuzz Tests

Create fuzz tests for system-level invariants not covered by Kontrol:
- **RX-005**: `testFuzz_adapter_rejectsUnauthorized` — random sender, expect revert
- **RX-006**: `testFuzz_swapTranslation_equivalence` — compare swap count increments
- **RX-007**: `testFuzz_mintTranslation_equivalence` — compare position registration
- **RX-009**: `testFuzz_unregisteredBurn_noop` — Burn on unknown position, state unchanged
- **RX-010**: `testFuzz_swapDedup_perBlock` — multiple swaps same block, at most 1 increment
